# 04 – Le navigateur, interprète des pages

Image simple
- Le serveur envoie du texte (HTML, CSS, JS).
- Le navigateur (Chrome, Firefox, Edge) traduit ce texte en images, boutons, couleurs.

Mini-démo choc
- Clic droit → « Afficher le code source » sur une page : montrer le texte brut vs l’affichage.
- Désactiver brièvement le CSS (via outils dev ou extension) pour voir la page « sans habits ».

Messages clés
- Créer un site = écrire le texte que le navigateur saura afficher.
- Si le code est cassé, le navigateur ne peut pas tout traduire (images manquantes, bloc mal placé).

Idée d’animation
- Avant/après : montrer une page avec styles, puis sans styles, et faire décrire les différences.
